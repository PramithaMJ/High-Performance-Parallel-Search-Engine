Performance Summary

The MPI version shows the best performance with an average query time of 78.3ms, followed by the OpenMP version at 124.6ms. The serial version is significantly slower at 365.2ms per query.

For indexing operations, we observe a similar pattern with MPI performing 3.95x faster than the serial version, while OpenMP shows a 2.67x improvement.

Scaling Efficiency

MPI demonstrates significantly better scaling across multiple nodes, with near-linear scaling up to 8 nodes, while OpenMP shows diminishing returns beyond 8 threads on a single node.

Memory Usage Summary

The serial version has the lowest memory footprint (245.6 MB), while the MPI version consumes the most memory (412.9 MB) due to its distributed nature. The OpenMP version falls in between at 283.7 MB.

All three versions use the same BM25 ranking algorithm and produce identical search results.
